Contact: Dr. Arash Sadrieh
Description: Defects in the electrical activity of heart cause ~12% of adult deaths in the developed world. Multi-scale computational modelling of the heart is now recognised as a powerful technique for interrogating the molecular and cellular basis of cardiac electrical activity. Computational approaches have allowed us to tease out important relations between parameters, which are not possible using current laboratory techniques. Until now, however, the application of computational methods has been limited by the enormous computational burden of current cardiac models. We overcome this computational limitation by applying General Purpose Graphical Processing Units (GPGPU). In this project we plan to automate the conversation of traditional CPU-based code to GPU-based code by implementing a code generator. We need to build a software system that receives a mathematical model expressed in a specific XML format (called CellML) and then generates an OpenCL solver for the model (We have a python module that generates C++ solvers, this module can be a good starting point).
Location: Victor Chang Cardiac Research Institute, Darlinghurst.
Desirable skills:: Python, Scons, Cmake, OpenCL, GPGPU, Code generation, Unit testing, working with large open-source software systems and little bit of C++(STL/boost)

  Victor Chang Institute: Web-based Holter recording data repository
Contact: Dr. Arash Sadrieh
Description: The electrocardiogram (ECG) is a time series of data representing electrical activity of the heart. For over 100 years it has been applied by cardiologists to diagnose many life threatening cardiac diseases. Using state-of-art machine learning and signal processing methods, we have shown that there is a wealth of useful new information that can be extracted from ECGs that could be used to improve the diagnosis and management of a range of cardiac disorders. To show the effectiveness of our methods we need access to a large database of ambulatory ECG recordings. We are in the process of establishing a large collaboration with hospitals across NSW to build an integrated web-based data repository for our study. We need to build a web-based system that will enable our (local/international) collaborators to upload relatively large data files (~4GB per patient).We estimate that the database will need to hold ~5000 data files. We want to run some back-end post processing on the data and provide collaborators the ability to query/download information from this dataset.
Location: Victor Chang Cardiac Research Institute, Darlinghurst.
Desirable skills:: Python, Django, Large time series datasets, Amazon S3/AWS EC2, Software integration mechanisms, Design patterns
