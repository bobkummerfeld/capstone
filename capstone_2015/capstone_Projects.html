  COMP Chapman: Chemical Labelling

Contact: Sandra Chapman -- sandra.chapman@sydney.edu.au

Description:
The NSW Work Health and Safety regulations require that all hazardous chemicals be appropriately labelled.   INFO Homeboy: Web interface and advanced camera control features.

Contact: Mark Hingston, markh@homeboy.com

Description:
The Problem:
We currently have Homeboy apps in the iOS and Android app stores that can view security footage from cameras, control cameras and add new cameras as required (see the links below). However some of our users don’t own an iPhone or Android phone or don’t always have access to one of these devices - so we’d love to add a web-based alternative. The web UI should contain some or all of the following abilities:

  • Authenticate users via OAuth 2 (there’s a tutorial for that in our docs)
  • Display a real time feed of events and alerts that occur at a location (eg. videos, messages, monitoring was turned on / off)
  • Turn on / off security monitoring at locations. Adjust camera and location settings.
  • Invite other users to join Homeboy
Plus, students should attempt to implement one or more of these advanced control features:
  • Advanced control
    Add new cameras to a user’s location. This involves stepping the user through the following process: turn on the camera's WiFi hotspot; connect the computer to the camera hotspot; ask the camera for the WiFi networks it can see; choose a network and send setup commands to the camera.
  • Intelligent battery conservation.
    A major selling point of Homeboy cameras is the battery life of the camera, but a major source of drain on the battery is that it 'wakes up' every minute to check what actions it should perform. By making the camera 'wake up’ less often we can increase the battery life of the camera. However the trade off is that the camera becomes less responsive to user input. The wake up time of a camera can already be controlled via a configuration mechanism. The Homeboy web UI should allow users to control the 'wake up' time of cameras. Thought should be given to the best way to communicate this tradeoff to the user.
    Further, the web UI should allow users to opt in to 'Intelligent battery conservation' - this would make dynamic wake up time changes based upon pre-existing Homeboy events such as: the user’s locale and current time; motion detection being turned on / off; or users of the system entering / leaving the home. Such a system might use simple or complex heuristics or even AI based methods to decide when to make the camera more or less responsive.
  • Auto configuration of 'Pet mode'.
    Pet mode allows Homeboy cameras to ignore motion events that were caused by a user's cat or dog. However, this setting can take a bit of trial and error to get right. If motion is detected by a Homeboy camera then the Web UI should allow users to choose an option like 'This is just my pet'. This information should be used to 'learn' the most appropriate setting for the user's home / pets. Students should attempt to design a system that converges on the right Pet mode setting with the least number of pet videos captured.

Links for additional information: Homeboy website | iPhone App | Android App | API Docs

Location: Client meetings to be co-ordinated with client.

  INFO Homeboy: Facial recognition for security camera footage..

Contact: Mark Hingston, markh@homeboy.com

Description:
The Problem:
Security systems can be a pain to turn on and off, and occasionally users can forget to turn off their system when they come home. If a user forgets to turn off their Homeboy camera then it will capture video of them entering their premises. We would like to use video analysis to recognise people in the Homeboy videos and train the system to know when a known user or an unexpected user has entered the home. This could influence our decisions on what notifications to send the user, or merely augment the users Homeboy experience.

What we need is a stand alone application that can take as an input a Homeboy video and identify some or all of the following:

  • Whether there are any people in the video, if so, how many.
  • Tag each person in the video using a pre-learned set of users and their videos or pictures.
  • Identify whether the person or persons in the video have been seen before at this location.
The Homeboy system should then be able to make use of the output data to make decisions and augment the users Homeboy experience.

Students may want to explore a number of different approaches to this problem, including variations in:

  • the enrolment method, i.e. how users are added to the facial recognition system. Perhaps including exploring the use of still photos, close up videos or ‘naturally occurring’ videos of users in their homes.
  • the algorithms used to identify faces and compare them with others.
  • the features extracted from faces.
Students should report on the effectiveness of each approach in identifying each of the criteria above. The final implementation may be some form of combination or hybrid of multiple approaches to the problem.

Links for additional information: Homeboy website | iPhone App | Android App | API Docs

Location: Client meetings to be co-ordinated with client.
